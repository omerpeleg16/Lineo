# LINEO UX Rules

Core user experience and interaction principles for LINEO landing page and product interfaces.

---

## 🎯 Core UX Principles

### 1. One Question Per Section
Every section must answer exactly **one** core question (e.g., *What problem exists?*, *How does LINEO solve it?*, *What can it generate?*, *How does it work?*).

### 2. Single Primary Action
Never introduce more than **one primary action (CTA)** per screen. Avoid competing calls-to-action that dilute user focus.

### 3. Concise Content
Keep body paragraphs under **four lines**. Ensure copy is punchy, structured, and effortless to digest.

### 4. Consistent Rhythm & Spacing
Maintain consistent vertical spacing between sections. Whitespace is an intentional layout element that lets the interface breathe.

### 5. 5-Second Clarity Rule
Users should understand the purpose and value proposition of each section within **5 seconds** of arriving at it.

### 6. Clear Visual Hierarchy
Visual hierarchy must always be unmistakable. Guide the user's attention naturally through scale, contrast, and element placement.

### 7. Show Before Explain
Always show the product, UI interface, or generated assets **before** explaining them with text.

### 8. Compare Before Explain
Use comparison (e.g., *Before vs. After*, *Mismatched Stock Assets vs. Unified Graphic Kit*) before explanation whenever possible.

### 9. Purposeful Visuals
Every visual element, graphic, and animation **must** support the narrative. Eliminate purely decorative or filler imagery.

### 10. Progressive Scroll Storytelling
Scrolling should reveal the product story progressively, creating a fluid, seamless narrative transition from top to bottom.
